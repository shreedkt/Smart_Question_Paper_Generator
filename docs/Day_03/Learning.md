**What is Application Flow?**
Application flow is the user's journey or the high-level business process. It is driven by the user's choices.
**What is Execution Flow?**
Execution flow is the strict, line-by-line path a computer takes to read and run your code. It is controlled by the programming logic itself.
**Why should every module have only one responsibility?**

**Why is main.py called the entry point?**
because it is the initial file that starts the execution of a Python application 
**What is a function skeleton?**
a structurally complete function definition that contains no actual execution logic
**Why do we use pass?**
pass is a null statement that acts as a placeholder for future code. It does nothing when executed, but prevents syntax or indentation errors in empty code blocks like functions, loops, or conditional statements.
